## Get user costs

Returns hourly cost records for the authenticated user within a time range, optionally filtered by namespace, app type, or app name.

**`POST /account/v1alpha1/costs`**

<ParamField body="startTime" type="string">
  Start of the time range in ISO 8601 format. Defaults to six months ago.
</ParamField>
<ParamField body="endTime" type="string">
  End of the time range in ISO 8601 format. Defaults to now.
</ParamField>
<ParamField body="namespace" type="string">
  Filter results to a specific Kubernetes namespace (e.g. `"ns-admin"`).
</ParamField>
<ParamField body="appType" type="string">
  Filter by application type (e.g. `"app"`, `"database"`).
</ParamField>
<ParamField body="appName" type="string">
  Filter by application name.
</ParamField>

### Example request

```bash
curl -X POST https://<region-domain>/account/v1alpha1/costs \
  -H "Authorization: Bearer <your-token>" \
  -H "Content-Type: application/json" \
  -d '{
    "startTime": "2024-05-01T00:00:00Z",
    "endTime":   "2024-06-01T00:00:00Z"
  }'
```

### Example response

```json
{
  "data": {
    "costs": {
      "2024-05-01T00:00:00Z": 12000,
      "2024-05-01T01:00:00Z": 11500
    }
  },
  "message": "successfully retrieved user costs"
}
```

<ResponseField name="data.costs" type="object">
  A map of ISO 8601 timestamps to cost amounts (in the platform's smallest monetary unit) for each hourly billing period.
</ResponseField>

## Get app costs

Returns per-app cost records within a time range. Supports filtering and pagination.

**`POST /account/v1alpha1/costs/app`**

<ParamField body="startTime" type="string">
  Start of the time range in ISO 8601 format.
</ParamField>
<ParamField body="endTime" type="string">
  End of the time range in ISO 8601 format.
</ParamField>
<ParamField body="namespace" type="string">
  Filter to a specific namespace.
</ParamField>
<ParamField body="appType" type="string">
  Filter by app type.
</ParamField>
<ParamField body="appName" type="string">
  Filter by app name.
</ParamField>
<ParamField body="orderID" type="string">
  Filter by a specific billing order ID.
</ParamField>
<ParamField body="page" type="integer">
  Page number (1-based). Defaults to `1`.
</ParamField>
<ParamField body="pageSize" type="integer">
  Number of records per page. Defaults to `10`.
</ParamField>

### Example request

```bash
curl -X POST https://<region-domain>/account/v1alpha1/costs/app \
  -H "Authorization: Bearer <your-token>" \
  -H "Content-Type: application/json" \
  -d '{
    "startTime": "2024-05-01T00:00:00Z",
    "endTime":   "2024-06-01T00:00:00Z",
    "namespace": "ns-admin",
    "page": 1,
    "pageSize": 20

    }
  ]
}
```

<ResponseField name="app_costs" type="array">

  <Expandable title="app cost fields">

    <ResponseField name="order_id" type="string">Unique billing order ID.</ResponseField>
    <ResponseField name="namespace" type="string">Kubernetes namespace.</ResponseField>
    <ResponseField name="amount" type="integer">Cost for this record in the smallest monetary unit.</ResponseField>
  </Expandable>
</ResponseField>

## Apply invoice

Creates an invoice from a list of payment IDs. The server validates that the referenced payments are eligible (not already invoiced) and returns the invoice record and the associated payment list.

**`POST /account/v1alpha1/invoice/apply`**

<ParamField body="paymentIDList" type="array" required>
  Array of payment IDs to include in this invoice.
</ParamField>
<ParamField body="detail" type="string" required>
  JSON-encoded string containing the invoice detail (billing address, company name, tax ID, etc.).
</ParamField>

### Example request

```bash
curl -X POST https://<region-domain>/account/v1alpha1/invoice/apply \
  -H "Authorization: Bearer <your-token>" \
  -H "Content-Type: application/json" \
  -d '{
    "paymentIDList": ["payment-id-1", "payment-id-2"],
    "detail": "{\"company\":\"Acme Corp\",\"taxID\":\"123456789\"}"
  }'
```

### Example response

```json
{
  "data": {
    "invoice": {
      "id": "invoice-id-new",
      "totalAmount": 200000000,
      "status": "PENDING"
    },
    "payments": [
      { "id": "payment-id-1", "amount": 100000000 },
      { "id": "payment-id-2", "amount": 100000000 }
    ]
  },
  "message": "successfully apply invoice"
}
```

<Note>
Returns `403 Forbidden` if none of the requested payment IDs are eligible for invoicing (e.g. all are already invoiced or the total amount is zero).
</Note>

<ResponseField name="data.invoice" type="object">
  The newly created invoice record.
</ResponseField>
<ResponseField name="data.payments" type="array">
  The payment records included in the invoice.
</ResponseField>

## Set invoice status

Updates the status of one or more invoices. This endpoint requires a separate invoice token (not the standard user JWT) and is intended for internal invoice processing integrations.

**`POST /account/v1alpha1/invoice/set-status`**

<ParamField body="invoiceIDList" type="array" required>
  Array of invoice IDs to update.
</ParamField>
<ParamField body="status" type="string" required>
  New status for the invoices. One of: `COMPLETED`, `REJECTED`, `PENDING`.
</ParamField>
<ParamField body="token" type="string" required>
  Invoice processing token (not the user JWT — see your platform configuration).
</ParamField>

### Example request

```bash
curl -X POST https://<region-domain>/account/v1alpha1/invoice/set-status \
  -H "Authorization: Bearer <your-token>" \
  -H "Content-Type: application/json" \
  -d '{
    "invoiceIDList": ["invoice-id-1"],
    "status": "COMPLETED",
    "token": "<invoice-token>"
  }'
```

### Example response

```json
{
  "message": "successfully set status invoice"
}
```

<Warning>
This endpoint validates against a server-side invoice token, not the standard user JWT. Calls with an incorrect invoice token receive `401 Unauthorized`.
</Warning>

## Get all-region consumption amount

Queries every configured region and returns the consumption amount per region as well as a global total. The calling user must be authenticated in the local region; the server fans out the query to remote regions using an internal service token.

**`POST /account/v1alpha1/costs/all-region-consumption`**

<ParamField body="startTime" type="string">Start of the time range.</ParamField>
<ParamField body="endTime" type="string">End of the time range.</ParamField>
<ParamField body="namespace" type="string">Limit to a specific namespace.</ParamField>
<ParamField body="appType" type="string">Limit to a specific app type.</ParamField>
<ParamField body="appName" type="string">Limit to a specific app name.</ParamField>

### Example request

```bash
curl -X POST https://<region-domain>/account/v1alpha1/costs/all-region-consumption \
  -H "Authorization: Bearer <your-token>" \
  -H "Content-Type: application/json" \
  -d '{
    "startTime": "2024-05-01T00:00:00Z",
    "endTime":   "2024-06-01T00:00:00Z"
  }'
```

### Example response

```json
{
  "regionAmount": {
    "us-west.cloud.sealos.io": 50000000,
    "ap-east.cloud.sealos.io": 30000000
  },
  "allAmount": 80000000
}
```

<ResponseField name="regionAmount" type="object">
  Map of region domain to consumption amount for that region.
</ResponseField>
<ResponseField name="allAmount" type="integer">
  Sum of consumption across all regions.
</ResponseField>
